# Plan: Playable 4-Player Cutthroat Cribbage (vs computer)

## Context

The repo currently ships a **discard trainer** (`src/CribbageTrainer.jsx` → `index.html`),
which analyzes/ranks discards but is not a playable game. The user wants a **separate page
they can navigate to and play a full game of cribbage against the computer** — scoped, for
now, to **4-player "cutthroat"** (every player for themselves, first to 121 wins). 4-player
partners mode is explicitly deferred.

This adds a new self-contained page `play.html` (source `src/CribbagePlay.jsx`) that reuses
the battle-tested scoring/pegging primitives from the trainer but adds the entire game loop:
deal → discard → cut → interactive pegging → the show/counting → score to 121 → rotate dealer.

User decisions captured:
- **Counting is a configurable setting**: *Auto-count* (default) or *Muggins* (you claim your
  own points; missed points are lost to an opponent).
- **Opponents' hands are hidden** during play, revealed at the show.

## Approach

A new single-component React page, built exactly like the trainer (React via CDN, `tsc` JSX
transpile, no framework). It re-implements the engine primitives by **copying them verbatim**
from `CribbageTrainer.jsx` — the two pages are independent self-contained HTML files and do
not share a module, which matches how the repo already works (the `engine/` scripts also
duplicate the math).

### React state (phase machine; `useReducer` recommended)
`phase: "deal"|"discard"|"cut"|"play"|"show"|"over"`, human is always seat 0.
- `dealerIdx` (rotates +1 each hand), `seats[4]` each `{ score, dealt, kept, discard, isAI }`
  (cards stay **suited** throughout), `crib[4]`, `starter`, `hisHeels`, `winner`, `log[]`.
- `peg` sub-state (during play): `hands[4]` (remaining suited cards/seat), `turn`, `count`,
  `pile[]` (ranks, for `pegScore`/`pegChoose`), `pileSuited[]` + `played[4]` (suited, display),
  `passes`, `lastPlayer`.
- `show` sub-state: `order = [dealer+1, dealer+2, dealer+3, dealer, "CRIB"]`, `step`, current
  breakdown.
- `settings: { counting: "auto" | "muggins" }`.

### Key functions
- **`startHand()`** — shuffle (`deckExcluding([])` + Fisher–Yates), deal 5/seat; compute the 3
  AI discards immediately; `phase="discard"`.
- **`aiDiscard(dealt5, seat)`** — for each of 5 throws: `handDetail(keptFour, dealt5).ev +
  sign*CRIB_VALUE[thrownRank]`, `sign=+1` if that seat is dealer else `-1`; pick best. ~30ms
  for 3 seats. `CRIB_VALUE` = the "your" crib-swing row from CLAUDE.md (per rank A–K).
- **`commitDiscard(humanIdx)`** — assemble `crib` from the 4 discards; `phase="cut"`.
- **`doCut()`** — `starter` = next undealt card; **if Jack → dealer +2 (his heels), check 121**;
  init `peg`; `phase="play"`.
- **Interactive pegging** — `playCard(seat, card)` (atomic: remove card, update count/pile,
  `pegScore` award + log + 121-check, handle 31 reset, advance turn, detect hand-exhaustion →
  last-card point → `phase="show"`), `passGo(seat)` (forced when no legal card; on 4 passes
  award go=1 to `lastPlayer` when `count!==31`, reset pile). A `useEffect` keyed on `peg.turn`
  drives it: human-with-legal-cards blocks for a tap; AI and all forced "go"s fire on a
  `setTimeout` (≈700ms) so the table animates — self-clocking loop.
- **Show sequencer** — `nextShowStep()` scores each seat's `kept` (then `crib` for dealer) via
  `scoreInto`, pegs it, **checks ≥121 after every award and stops immediately** (counting-order
  short-circuit). Advances on a "Continue" tap so the human can read each breakdown (`CatBars`).
- **`checkWin(seat)`** — called after every score mutation; first to ≥121 → `winner`,
  `phase="over"`. "Play again" → rotate dealer, re-deal.

### Counting setting (auto vs muggins)
- **Auto** (default): the show sequencer tallies and pegs every hand automatically.
- **Muggins**: AI hands always auto-count; but when it's the **human's** turn to count their own
  hand (and the crib if dealer), show the cards and a **claim input** (tap the number / +/−).
  Award `min(claim, actual)`; if `claim < actual`, the missed points are "mugginsed" to the next
  opponent in counting order (simplified one-directional muggins — the human only polices their
  own count). Over-claims are corrected down to `actual` (no phantom points).
- A small **settings control** on the page toggles `counting` (Auto / Muggins).

### New UI
- **`CardBack`** (new, small): face-down card (wood-grain gradient) for opponents during
  discard/cut/play. Opponents reveal face-up `Card`s only at the show.
- Screens: **deal/seat** (4 score chips + dealer chip + per-seat peg position; "Deal"),
  **discard** (your hand face-up, tap one; AI face-down), **cut** (flip starter; his-heels

  (winner + Play again). Reuse `Card` `phase`/`dim`/`onClick`, `PegTrack`, `CatBars`,

## Correctness pitfalls (must test)
1. **Go vs 31 double-count:** `pegScore` already returns +2 on 31; award go/last-card +1 only
   when `count !== 31`; after a 31 reset set `lastPlayer=-1` so end-of-hand last-card doesn't
   double-pay. Mirror `playPegging` exactly.
2. **121 short-circuit in the show:** stop the instant any seat hits 121, before counting the
   next player or the crib (a non-dealer to the dealer's left can peg out first).
3. **His heels = 2** (not 1), applied at the cut, 121-checked.
4. **Suits preserved through pegging:** only the throwaway `pile`/`hand` arrays passed to
   `pegScore`/`pegChoose` are ranks; `kept`/`crib`/`played`/`pileSuited`/`starter` stay suited
   (flush + nobs scoring depends on real suits). Map AI's chosen rank back to a specific card.
5. **Forced-go detection** on the real remaining hand, not the original four.
